# SAFT-PT Automation

![Python](https://img.shields.io/badge/python-3.10%2B-blue)
![License](https://img.shields.io/github/license/cunhabarbosa/saft-pt-automation)
![Repo size](https://img.shields.io/github/repo-size/cunhabarbosa/saft-pt-automation)
![Last commit](https://img.shields.io/github/last-commit/cunhabarbosa/saft-pt-automation)

Automate the detection, validation, emailing (via Outlook Desktop App), and organization of **SAFT-PT and e-Fatura files**.

This tool scans a folder (e.g. Desktop), detects the latest billing period, validates required files, opens a pre-filled Outlook email with attachments, and optionally moves files to structured folders.

Built for internal accounting workflows but reusable for similar file automation tasks.

---

## Features

- Automatically detect billing period from filenames (`YYYY-MM`)
- Validate required files before sending
- Open Outlook email draft with attachments
- Organize files into structured folders:
  - e-Fatura comprovativos
  - Mapa recapitulativo (mensal)
  - SAFT-PT
- Prevent overwriting existing files
- Configurable paths and keywords via `config.json`

---

## Motivation

Managing monthly accounting files manually (SAFT, PDFs, XMLs) is repetitive and error-prone.

This tool ensures:
- all required files are present
- consistent naming and organization
- faster workflow with minimal user interaction

---

## Workflow

1. Scan folder for files  
2. Detect latest period (`YYYY-MM`)  
3. Validate required files  
4. Open Outlook email with attachments  
5. Ask user whether to move files  
6. Move files to destination folders

```json
    "efactura": "e-factura comprovativos",

    "saft": "SAFT-PT"
  },
  "keywords": ["Porto", "S J Madeira"]

```

## Expected file naming

The tool expects files following this pattern:
```text
YYYY-MM.pdf
YYYY-MM Porto.pdf
YYYY-MM Porto.xml
YYYY-MM S J Madeira.pdf
YYYY-MM S J Madeira.xml
```

Example:
```text
2026-04.pdf
2026-04 Porto.pdf
2026-04 Porto.xml
2026-04 S J Madeira.pdf
2026-04 S J Madeira.xml
```

## Requirements
- Python 3.10+
- Microsoft Outlook Desktop (required for email creation)
- Windows OS (uses `win32com`)

---

## Notes
- Files must follow the expected naming convention
- The tool will stop if required files are missing
- Existing files in destination folders will not be overwritten
- Email is opened in Outlook — the user must manually send it

---

## Executable Usage
You can build a standalone executable:
```bash
pyinstaller --onefile --name saft-pt-automation main.py
```

Then run:
```bash
dist/saft-pt-automation.exe
```

Make sure `config.json` is in the same folder as the executable (same directory as the `.exe` file).
